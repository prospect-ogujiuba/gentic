# SWE Plan

Create a compact plan before implementation.

## Workflow

1. Define the problem, outcome, constraints, and non-goals.
2. Design the likely components, data/control flow, risks, and unknowns.
3. Slice the smallest honest vertical change that can prove value.
4. Write a concrete Definition of Done.
5. Choose focused verification commands or manual checks.

## Success criteria

- The plan is small enough to implement surgically.
- Acceptance criteria and verification evidence are explicit.
- No legacy SOP command namespace is introduced.
