Plan the SWE work for: $ARGUMENTS

Produce a concise plan with these sections:

1. Define — problem, user-visible outcome, constraints, and non-goals.
2. Design — affected components, data/control flow, risks, and unknowns.
3. Slice — the smallest honest vertical slice that proves value.
4. Definition of Done — concrete acceptance criteria.
5. Verification Plan — focused compile/run/test/manual checks and expected evidence.

Keep scope surgical. Do not implement yet.
